# Báo cáo triển khai hệ thống log tin nhắn Telegram ra Google Sheets

## 1. Mục tiêu của task

Xây dựng một tiến trình tự động ghi lại tin nhắn trong nhóm Telegram vào Google Sheets theo thời gian gần realtime.

### 1.1. Phạm vi yêu cầu
Hệ thống chỉ ghi:
1. Tin nhắn của người dùng thật
2. Chỉ các thành viên được chỉ định, ví dụ bạn và anh Cường
3. Nội dung text hoặc caption hợp lệ

Hệ thống không ghi:
1. Tin nhắn từ bot
2. Sự kiện hệ thống của Telegram
3. Tin nhắn của người ngoài danh sách cho phép
4. Tin nhắn trùng lặp

### 1.2. Kết quả mong muốn
Dữ liệu được ghi vào Google Sheets theo các cột:

**STT | Ngày giờ | Nội dung | Tên người gửi | Message ID**

Trong đó:
- `STT`: số thứ tự
- `Ngày giờ`: thời điểm gửi theo giờ Việt Nam
- `Nội dung`: text hoặc caption
- `Tên người gửi`: tên Telegram của người gửi
- `Message ID`: mã tin nhắn để chống trùng

---

## 2. Kiến trúc tổng thể

### 2.1. Thành phần hệ thống
Hệ thống gồm 3 phần chính:

1. **Telegram Group**
   - Chứa người dùng và bot
   - Là nguồn phát sinh tin nhắn

2. **Python Logger**
   - Nhận tin nhắn từ Telegram Bot API
   - Lọc đúng group
   - Lọc đúng người dùng
   - Loại bỏ bot
   - Ghi dữ liệu sang Google Sheets

3. **Google Sheets**
   - Là nơi hiển thị log theo thời gian thực gần realtime

### 2.2. Sơ đồ luồng xử lý

```text
Người dùng nhắn tin trong Telegram Group
        ↓
Telegram Bot nhận message
        ↓
Python logger xử lý message
        ↓
Kiểm tra group, user, bot, nội dung, chống trùng
        ↓
Append dữ liệu vào Google Sheets
```

---

## 3. Quyết định kiến trúc đã chốt

Trong quá trình thiết kế, có một số phương án được cân nhắc.

### 3.1. Phương án không chọn
Dùng chung một bot cho cả:
- trả lời tin nhắn bằng OpenClaw
- ghi log vào Google Sheets

Phương án này gây lỗi `409 Conflict` vì Telegram không cho nhiều tiến trình cùng dùng `getUpdates` trên một bot token.

### 3.2. Phương án đã chọn
Tách thành 2 bot riêng:

1. **Bot trả lời**
   - Dùng cho OpenClaw
   - Trả lời câu hỏi trong nhóm

2. **Bot đọc / logger**
   - Dùng riêng để đọc tin nhắn
   - Ghi dữ liệu ra Google Sheets

Đây là phương án ổn định và dễ bảo trì nhất.

## 4. Chuẩn bị môi trường

### 4.1. Thư mục dự án
Tạo thư mục làm việc, ví dụ:

```text
C:\Users\duynk\telegram_logger
```

Trong thư mục này chứa:
- `bot_logger.py`
- `debug_telegram.py`
- `telegram-logger-key.json`
- thư mục `venv` sau khi tạo môi trường ảo

### 4.2. Cài Python
Yêu cầu máy có Python cài sẵn và hoạt động được trong terminal.

Kiểm tra bằng:

```bash
python --version
```

Nếu không chạy được, có thể dùng:

```bash
py --version
```

### 4.3. Tạo môi trường ảo `venv`
Trong PowerShell hoặc CMD, di chuyển tới thư mục project:

```bash
cd C:\Users\duynk\telegram_logger
```

Tạo `venv`:

```bash
py -m venv venv
```

Hoặc:

```bash
python -m venv venv
```

### 4.4. Cài thư viện Python
Không cần activate nếu không muốn. Có thể gọi thẳng Python trong `venv`:

```bash
.\venv\Scripts\python.exe -m pip install python-telegram-bot gspread google-auth pytz
```

Các thư viện sử dụng:
- `python-telegram-bot`
- `gspread`
- `google-auth`
- `pytz`

## 8. Lấy thông tin Telegram cần thiết

Cần có đủ 3 loại thông tin:

1. `BOT_TOKEN`
2. `GROUP_ID`
3. `USER_ID` của từng người được phép ghi log

### 8.1. Tạo file debug để đọc thông tin Telegram
Tạo file `debug_telegram.py` với nhiệm vụ in ra:
- `chat.id`
- `chat.title`
- `user.id`
- `user.full_name`
- `user.is_bot`
- `message.message_id`
- `message.text`

### 8.2. Chạy file debug
Chạy bằng:

```bash
.\venv\Scripts\python.exe .\debug_telegram.py
```

Sau đó nhắn thử trong group để lấy:
- `GROUP_ID`
- `USER_ID` của bạn
- `USER_ID` của anh Cường

## 9. Thiết kế logic lọc dữ liệu

### 9.1. Lọc theo đúng group
Chỉ xử lý tin nhắn nếu:

`chat.id == GROUP_ID`

### 9.2. Loại bỏ bot
Chỉ xử lý khi:

`user.is_bot == False`

### 9.3. Lọc đúng người dùng
Chỉ cho phép ghi log nếu:

`user.id` nằm trong `ALLOWED_USER_IDS`

### 9.4. Chỉ lấy nội dung hợp lệ
Ưu tiên:
1. `message.text`
2. `message.caption`

Nếu không có nội dung thì bỏ qua.

### 9.5. Chống trùng
Kiểm tra `message.message_id` đã có trong cột `Message ID` hay chưa.

Nếu đã có thì không ghi thêm.

## 10. File logger chính

### 10.1. Tên file
`bot_logger.py`

### 10.2. Nhiệm vụ
- kết nối Telegram
- kết nối Google Sheets
- nhận tin nhắn mới
- lọc dữ liệu
- ghi một dòng vào Google Sheets

### 10.3. Cấu hình chính trong code

```python
BOT_TOKEN = "..."
GROUP_ID = -100...
ALLOWED_USER_IDS = {
    111111111,
    222222222
}
GOOGLE_SHEET_NAME = "Telegram Chat Log"
WORKSHEET_NAME = "chat_log"
SERVICE_ACCOUNT_FILE = "telegram-logger-key.json"
```

### 10.4. Luồng xử lý trong `handle_message`
1. Kiểm tra message, chat, user có tồn tại
2. Kiểm tra đúng group
3. Bỏ bot
4. Kiểm tra đúng user
5. Lấy nội dung
6. Kiểm tra trùng
7. Chuyển thời gian sang giờ Việt Nam
8. Ghi một dòng vào Google Sheets

## 12. Vấn đề đã gặp trong quá trình triển khai và cách xử lý

### 12.1. Python chưa cài hoặc chưa vào PATH
Triệu chứng:
- `python` không chạy
- `python -m venv venv` lỗi

Cách xử lý:
- cài Python
- hoặc dùng `py -m venv venv`

### 12.2. Chưa có thư mục `venv`
Triệu chứng:
- `Activate.ps1` không tìm thấy

Cách xử lý:
- tạo lại `venv` bằng `py -m venv venv`

### 12.3. PowerShell không activate được
Triệu chứng:
- lỗi `.\venv\Scripts\Activate.ps1`

Cách xử lý:
- không cần activate
- gọi trực tiếp:

```bash
.\venv\Scripts\python.exe .\bot_logger.py
```

### 12.4. Xung đột `409 Conflict` với Telegram
Triệu chứng:
- OpenClaw báo lỗi `terminated by other getUpdates request`

Nguyên nhân:
- nhiều tiến trình cùng dùng một bot token

Cách xử lý:
- tách bot trả lời và bot logger thành 2 bot riêng

### 12.5. Google Sheets không ghi được
Nguyên nhân phổ biến:
- sai tên sheet
- sai worksheet
- chưa share cho service account
- sai đường dẫn JSON key

### 12.6. Logger ghi cả tin nhắn bot
Nguyên nhân:
- chưa lọc `user.is_bot`
- đưa nhầm bot ID vào danh sách user được phép

### 12.7. Logger ghi trùng
Nguyên nhân:
- chưa kiểm tra `Message ID`

Cách xử lý:
- dùng cột `Message ID` để chống trùng

## 13. Khởi chạy lại sau khi tắt máy tính

Đây là phần quan trọng cho vận hành thực tế.

### 13.1. Cách chạy lại trên Windows
Sau khi bật máy, mở PowerShell hoặc CMD.

Di chuyển tới thư mục project:

```bash
cd C:\Users\duynk\telegram_logger
```

### 13.2. Cách khởi chạy đơn giản nhất
Không cần activate môi trường ảo. Chạy trực tiếp:

```bash
.\venv\Scripts\python.exe .\bot_logger.py
```

Đây là cách được khuyến nghị vì:
- ít lỗi hơn activate
- không bị vướng execution policy của PowerShell
- rõ ràng và dễ nhớ

### 13.3. Nếu muốn activate rồi mới chạy
Trong PowerShell:

```bash
Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass
.\venv\Scripts\Activate.ps1
python .\bot_logger.py
```

Tuy nhiên cách này không cần thiết nếu chỉ muốn chạy nhanh.

### 13.4. Câu lệnh khởi chạy khuyến nghị
Câu lệnh chuẩn để chạy lại sau khi tắt máy:

```bash
cd C:\Users\duynk\telegram_logger
.\venv\Scripts\python.exe .\bot_logger.py
```

## 15. Khuyến nghị vận hành

### 15.1. Không dùng chung bot token cho nhiều tiến trình
Mỗi bot token chỉ nên để một tiến trình polling sử dụng.

### 15.2. Luôn dùng `Message ID` để chống trùng
Điều này giúp hệ thống ổn định khi restart.

### 15.3. Tách riêng bot logger và bot trả lời
Giúp:
- tránh xung đột
- dễ bảo trì
- dễ mở rộng

### 15.4. Không chỉnh sửa trực tiếp file JSON key lung tung
Vì đây là file xác thực quan trọng.

### 15.5. Nên backup code và file cấu hình
Backup tối thiểu:
- `bot_logger.py`
- `debug_telegram.py`
- `requirements.txt` nếu có
- hướng dẫn vận hành
- không backup công khai file JSON key
