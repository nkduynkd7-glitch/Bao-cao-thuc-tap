# BÁO CÁO: JOIN DOMAIN VÀO SAMBA DC VÀ CẤU HÌNH MULTI LOGIN RDP
 
**Môi trường thực hiện:**
- Samba DC: `172.16.20.97` | Domain: `lab.local` | NetBIOS: `LAB`
- Windows Server (Member): `172.16.20.111`
- Hostname máy Windows: `WIN-MUCIO8NRG4E`
 
---
 
## PHẦN 1: JOIN DOMAIN VÀO SAMBA DC
 
### 1.1. Giới thiệu
 
**Join Domain** là quá trình đưa một máy tính vào hệ thống quản lý tập trung (Active Directory / Samba DC). Sau khi join, máy tính sẽ chịu sự quản lý của Domain Controller (DC) và có thể:
 
- Xác thực user thông qua DC (không cần tạo account riêng trên từng máy)
- Nhận và áp dụng Group Policy từ DC
- Truy cập tài nguyên chia sẻ trong domain (shared folder, máy in,...)
- Đăng nhập một lần dùng được nhiều dịch vụ (SSO - Single Sign-On)
 
### 1.2. Kiểm tra thông tin Samba DC
 
Trước khi join, xác định tên domain bằng lệnh sau trên máy Samba DC:
 
```bash
samba-tool domain info 172.16.20.97
```
 
**Kết quả:**
```
Forest           : lab.local
Domain           : lab.local
Netbios domain   : LAB
DC name          : dc-nvc.lab.local
DC netbios name  : DC-NVC
Server site      : Default-First-Site-Name
```
 
### 1.3. Cấu hình DNS trên Windows Server
 
Máy Windows phải trỏ DNS về Samba DC để phân giải được tên domain.
 
**Bước 1:** Kiểm tra tên card mạng
 
```powershell
Get-NetAdapter
```
 
> Kết quả: Tên card mạng là **Ethernet**
 
**Bước 2:** Đặt DNS Server trỏ về Samba DC
 
```powershell
Set-DnsClientServerAddress -InterfaceAlias "Ethernet" -ServerAddresses "172.16.20.97"
```
 
**Bước 3:** Flush DNS cache và kiểm tra
 
```powershell
ipconfig /flushdns
nslookup lab.local
```
 
**Kết quả mong đợi:**
```
Server:  dc-nvc.lab.local
Address: 172.16.20.97
 
Name:    lab.local
Address: 172.16.20.97
```
 
### 1.4. Thực hiện Join Domain
 
```powershell
Add-Computer -DomainName "lab.local" -Credential "LAB\Administrator" -Restart
```
 
- Nhập **Username:** `LAB\Administrator`
- Nhập **Password:** password của Administrator trên Samba DC
 
Máy sẽ tự động **restart** sau khi join thành công.
 
### 1.5. Xác nhận Join Domain thành công
 
Sau khi restart, màn hình đăng nhập hiển thị **"Sign in to: LAB"** xác nhận đã join domain thành công.
 
Kiểm tra thêm bằng PowerShell:
 
```powershell
(Get-WmiObject Win32_ComputerSystem).Domain
# Kết quả: lab.local
 
nltest /dsgetdc:lab.local
```

## PHẦN 2: QUẢN LÝ USER TRÊN SAMBA DC
 
### 2.1. Xem danh sách user
 
```bash
samba-tool user list
```
 
**Kết quả:**
```
Guest
cuongnv
ldapbind
Administrator
tunv
krbtgt
```
 
### 2.2. Tạo user mới
 
```bash
# Tạo user duynk
samba-tool user create duynk
 
# Tạo user quyentv
samba-tool user create quyentv
 
# Tạo user vuongdv
samba-tool user create vuongdv
```
 
> **Lưu ý:** Password phải đủ mạnh — ít nhất 8 ký tự, có chữ hoa, chữ thường, số và ký tự đặc biệt. Ví dụ: `User@123456`
 
### 2.3. Reset password user
 
```bash
samba-tool user setpassword Administrator
```

## PHẦN 3: CẤU HÌNH MULTI LOGIN RDP
 
### 3.1. Giới thiệu
 
Mặc định Windows Server chỉ cho phép **2 session RDP** đồng thời (dành cho quản trị viên). Để cho phép nhiều user đăng nhập cùng lúc, cần cài đặt **Remote Desktop Services (RDS)** và cấu hình licensing.
 
### 3.2. Cài đặt RDS Role
 
```powershell
# Cài Remote Desktop Session Host
Install-WindowsFeature RDS-RD-Server -IncludeManagementTools
 
# Cài Remote Desktop Licensing
Install-WindowsFeature RDS-Licensing -IncludeManagementTools
```
 
> **Lưu ý:** Sau khi cài xong, hệ thống yêu cầu restart để hoàn tất.
 
### 3.3. Cấu hình Licensing qua Registry
 
```powershell
# Trỏ license server về chính máy này
$server = $env:COMPUTERNAME
Set-ItemProperty -Path 'HKLM:\SYSTEM\CurrentControlSet\Services\TermService\Parameters\LicenseServers' `
    -Name "LicenseServers" -Value $server
 
# Đặt mode Per Device
Set-ItemProperty -Path 'HKLM:\SYSTEM\CurrentControlSet\Control\Terminal Server\RCM\Licensing Core' `
    -Name "LicensingMode" -Value 2
```
 
### 3.4. Cấu hình qua Group Policy
 
Mở Group Policy Editor:
 
```
Win + R → gpedit.msc
```
 
Vào theo đường dẫn:
 
```
Computer Configuration
  └── Administrative Templates
        └── Windows Components
              └── Remote Desktop Services
                    └── Remote Desktop Session Host
                          └── Licensing
                                ├── "Use the specified RD license servers"
                                │     → Enable → Nhập: WIN-MUCIO8NRG4E
                                └── "Set the Remote Desktop licensing mode"
                                      → Enable → Chọn: Per Device
```
 
### 3.5. Cấu hình cho phép nhiều session
 
```powershell
# Bật RDP
Set-ItemProperty -Path 'HKLM:\System\CurrentControlSet\Control\Terminal Server' `
    -Name "fDenyTSConnections" -Value 0
 
# Cho phép nhiều user login cùng lúc
Set-ItemProperty -Path 'HKLM:\System\CurrentControlSet\Control\Terminal Server' `
    -Name "fSingleSessionPerUser" -Value 0
 
# Tắt Network Level Authentication (NLA) để tránh lỗi CredSSP
Set-ItemProperty -Path 'HKLM:\System\CurrentControlSet\Control\Terminal Server\WinStations\RDP-Tcp' `
    -Name "UserAuthentication" -Value 0
 
# Tắt SecurityLayer
Set-ItemProperty -Path 'HKLM:\System\CurrentControlSet\Control\Terminal Server\WinStations\RDP-Tcp' `
    -Name "SecurityLayer" -Value 0
 
# Fix lỗi CredSSP
REG ADD HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\Policies\System\CredSSP\Parameters `
    /v AllowEncryptionOracle /t REG_DWORD /d 2 /f
 
# Cho phép qua Firewall
Enable-NetFirewallRule -DisplayGroup "Remote Desktop"
 
# Restart dịch vụ RDP
Restart-Service TermService -Force
```
 
### 3.6. Thêm user vào nhóm Remote Desktop Users
 
```powershell
Add-LocalGroupMember -Group "Remote Desktop Users" -Member "LAB\duynk"
Add-LocalGroupMember -Group "Remote Desktop Users" -Member "LAB\cuongnv"
Add-LocalGroupMember -Group "Remote Desktop Users" -Member "LAB\tunv"
Add-LocalGroupMember -Group "Remote Desktop Users" -Member "LAB\quyentv"
Add-LocalGroupMember -Group "Remote Desktop Users" -Member "LAB\vuongdv"
```
 
### 3.7. Restart máy
 
```powershell
Restart-Computer
```
 
### 3.8. Kiểm tra sau khi cấu hình
 
```powershell
# Xem các session đang active
query session
 
# Hoặc
qwinsta
```
 
**Kết quả mong đợi:**
```
SESSIONNAME    USERNAME    ID    STATE
console        Admin        0    Active
rdp-tcp#1      tunv         1    Active
rdp-tcp#2      cuongnv      2    Active
rdp-tcp#3      quyentv      3    Active
```

## PHẦN 6: XỬ LÝ LỖI THƯỜNG GẶP

**Lỗi CredSSP trên máy thật khác! Chạy lệnh này trên máy thật bị lỗi (với quyền Administrator)**
```bash
REG ADD HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\Policies\System\CredSSP\Parameters /v AllowEncryptionOracle /t REG_DWORD /d 2 /f
```

 
| Lỗi | Nguyên nhân | Giải pháp |
|-----|-------------|-----------|
| `nslookup` timeout | DNS chưa trỏ về DC | Đặt DNS = `172.16.20.97` |
| "The user name or password is incorrect" | Sai credentials | Reset password bằng `samba-tool user setpassword` |
| CredSSP encryption oracle remediation | Không khớp phiên bản bảo mật | Chạy lệnh fix CredSSP cả 2 phía, tắt NLA |
| Chỉ login được 2 session | Chưa cài RDS | Cài `RDS-RD-Server` và `RDS-Licensing` |
| Không tìm thấy domain | Tên DNS sai | Kiểm tra lại `nslookup lab.local` |
