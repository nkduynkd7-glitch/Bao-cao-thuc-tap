## 1. Roaming Profile là gì?

**Roaming Profile** là tính năng của Windows Domain cho phép lưu toàn bộ profile người dùng (Desktop, Documents, Settings, dữ liệu ứng dụng...) lên một File Server trung tâm. Khi người dùng đăng nhập vào **bất kỳ máy nào** trong domain, Windows sẽ tự động tải profile từ server về — giúp trải nghiệm làm việc hoàn toàn giống nhau trên mọi thiết bị.

### Những gì được sync

| Loại dữ liệu | Sync được |
|---|---|
| Desktop (file, shortcut) | ✅ Có |
| Documents, Downloads | ✅ Có |
| Cài đặt ứng dụng (Chrome bookmarks, Zalo settings...) | ✅ Có |
| Wallpaper, theme | ✅ Có |
| Dữ liệu ứng dụng (AppData) | ✅ Có |
| Bản thân ứng dụng (file cài đặt) | ❌ Không — phải cài trên từng máy |

## 4. Bước 1: Tạo Shared Folder trên File Server

> ⚠️ **Thực hiện trên máy 172.16.20.30**
> 👤 **Tài khoản: AD\Administrator (Run as Administrator)**

### 4.1. Tạo thư mục Profiles

```powershell
# Tạo thư mục lưu Roaming Profile
New-Item -Path "C:\Profiles" -ItemType Directory
```

### 4.2. Tạo Shared Folder ẩn

```powershell
# Dấu $ cuối tên share để ẩn thư mục khỏi Network Browser
# User không thấy khi duyệt mạng nhưng vẫn truy cập được qua đường dẫn trực tiếp
New-SmbShare -Name "Profiles$" -Path "C:\Profiles" -FullAccess "Everyone"
```

### 4.3. Phân quyền NTFS

```powershell
$acl = Get-Acl "C:\Profiles"
$rule = New-Object System.Security.AccessControl.FileSystemAccessRule(
    "Everyone",
    "FullControl",
    "ContainerInherit,ObjectInherit",
    "None",
    "Allow"
)
$acl.SetAccessRule($rule)
Set-Acl "C:\Profiles" $acl
```

### 4.4. Kiểm tra share đã tạo thành công

```powershell
Get-SmbShare -Name "Profiles$"
```

**Kết quả mong đợi:**
```
Name      ScopeName Path        Description
----      --------- ----        -----------
Profiles$ *         C:\Profiles
```

## 5. Bước 2: Set Profile Path cho user trên Domain

> ⚠️ **Thực hiện trên máy 172.16.20.30**
> 👤 **Tài khoản: AD\Administrator (Run as Administrator)**

### 5.1. Set Profile Path cho maitt01

```powershell
# Trỏ profile path của maitt01 về File Server
net user maitt01 /profilepath:"\\172.16.20.30\Profiles$\maitt01" /domain
```

### 5.2. Kiểm tra Profile Path đã được set

```powershell
net user maitt01 /domain
```

**Kết quả mong đợi — tìm dòng User profile:**
```
User profile    \\172.16.20.30\Profiles$\maitt01
```

> **Lưu ý:** Windows sẽ tự động thêm `.V6` vào tên thư mục khi user đăng nhập lần đầu
> (`.V6` = định dạng Roaming Profile của Windows 10/11/Server 2016+)

## 6. Bước 3: Cấu hình GPO Roaming Profile

> ⚠️ **Thực hiện trên máy 172.16.20.30 hoặc bất kỳ máy nào có GPMC**
> 👤 **Tài khoản: AD\Administrator**

### 6.1. Mở Group Policy Management

```powershell
gpmc.msc
```

### 6.2. Tạo GPO mới

```
Group Policy Management
  └── Forest: ad.hpadirect.vn
        └── Domains
              └── ad.hpadirect.vn
                    └── Group Policy Objects
                          └── Chuột phải → New
                                → Đặt tên: "RoamingProfile"
                                → OK
```

### 6.3. Edit GPO RoamingProfile

Chuột phải vào **"RoamingProfile"** → **Edit** → vào:

```
Computer Configuration
  └── Policies
        └── Administrative Templates
              └── System
                    └── User Profiles
                          └── "Set roaming profile path for all users
                               logging onto this computer"
                                → Double click
                                → Chọn: Enable
                                → Value: \\172.16.20.30\Profiles$\%USERNAME%
                                → Apply → OK
```

> **Lưu ý:** `%USERNAME%` là biến tự động — Windows tự thay bằng tên user đang đăng nhập.
> Ví dụ: maitt01 đăng nhập → `\\172.16.20.30\Profiles$\maitt01`

### 6.4. Link GPO vào Domain

```
Group Policy Management
  └── ad.hpadirect.vn
        └── Chuột phải → "Link an Existing GPO"
              → Chọn: "RoamingProfile"
              → OK
```

### 6.5. Cập nhật Group Policy

> ⚠️ **Thực hiện trên CẢ 2 MÁY (.30 và .111)**
> 👤 **Tài khoản: AD\Administrator (Run as Administrator)**

```powershell
gpupdate /force
```

## 7. Bước 4: Xóa Local Profile cũ

> ⚠️ **Thực hiện trên CẢ 2 MÁY (.30 và .111)**
> 👤 **Tài khoản: AD\Administrator (Run as Administrator)**
> ⚠️ **Phải đăng xuất maitt01 trước khi thực hiện bước này!**

Nếu máy đã từng đăng nhập bằng `maitt01` trước đó, sẽ có **local profile cũ** tồn tại. Windows sẽ ưu tiên dùng local profile thay vì Roaming Profile → cần xóa đi.

### 7.1. Xóa Registry Profile cũ

```powershell
$regPath = "HKLM:\SOFTWARE\Microsoft\Windows NT\CurrentVersion\ProfileList"
Get-ChildItem $regPath | ForEach-Object {
    $prof = Get-ItemProperty $_.PSPath
    if ($prof.ProfileImagePath -like "*maitt01*") {
        Write-Host "Tìm thấy: $($prof.ProfileImagePath)"
        Remove-Item $_.PSPath -Recurse -Force
        Write-Host "Đã xóa registry profile!" -ForegroundColor Green
    }
}
```

### 7.2. Xóa thư mục Local Profile cũ

```powershell
# Lấy quyền sở hữu
takeown /f "C:\Users\maitt01" /r /d y
icacls "C:\Users\maitt01" /grant administrators:F /t

# Xóa thư mục
Remove-Item "C:\Users\maitt01" -Recurse -Force -ErrorAction SilentlyContinue
```

### 7.3. Hoặc xóa qua GUI (dễ hơn)

```
Chuột phải "This PC" → Properties
  → Advanced system settings
    → Tab "Advanced"
      → User Profiles → Settings
        → Chọn "AD\maitt01" → Delete → Yes
        → OK
```

> **Lưu ý:** Nút Delete chỉ active khi maitt01 **không đang đăng nhập**.

### 7.4. Restart máy

```powershell
Restart-Computer -Force
```

## 9. Xử lý lỗi thường gặp

### 9.1. Roaming Profile hiện N/A

**Nguyên nhân:** Profile path chưa được set hoặc bị sai.

**Giải pháp:**
> 👤 **Tài khoản: AD\Administrator (Run as Administrator)**

```powershell
net user maitt01 /profilepath:"\\172.16.20.30\Profiles$\maitt01" /domain
net user maitt01 /domain
```

---

### 9.2. Vẫn dùng Local Profile dù đã set Roaming Profile

**Nguyên nhân:** Local Profile cũ vẫn còn tồn tại, Windows ưu tiên dùng local.

**Giải pháp:** Xóa local profile cũ theo Bước 4.

---

### 9.3. Lỗi đồng hồ không đồng bộ khi gpupdate

**Triệu chứng:**
```
Windows was unable to determine whether new Group Policy settings...
computer's clock is not synchronized with the clock of one of the domain controllers
```

**Giải pháp:**
> 👤 **Tài khoản: AD\Administrator (Run as Administrator)**

```powershell
# Đồng bộ timezone Việt Nam
Set-TimeZone -Id "SE Asia Standard Time"

# Bật Windows Time service
Start-Service w32tm
Set-Service w32tm -StartupType Automatic

# Sync với DC
w32tm /config /manualpeerlist:"172.16.60.110" /syncfromflags:manual /update
w32tm /resync /force

# Chạy lại gpupdate
gpupdate /force
```

---

### 9.4. Lỗi Access Denied khi tạo SMB Share

**Nguyên nhân:** Đang chạy bằng tài khoản không đủ quyền (maitt01 thay vì Administrator).

**Giải pháp:** Mở PowerShell → **Run as Administrator** → chạy lại lệnh.

---

### 9.5. Profile không sync khi đăng nhập máy thứ 2

**Nguyên nhân:** Chưa Sign Out đúng cách trên máy thứ nhất.

**Giải pháp:**
- Luôn **Sign Out** thay vì tắt máy hoặc lock màn hình
- Chờ ít nhất 30 giây sau Sign Out để profile upload xong

## 10. Lưu ý quan trọng

### 10.1. Không đăng nhập đồng thời trên 2 máy

> ⚠️ **Cảnh báo:** Nếu đăng nhập `maitt01` trên cả 2 máy cùng lúc, khi Sign Out sẽ xảy ra **Profile Conflict** — máy Sign Out sau sẽ ghi đè profile của máy Sign Out trước → có thể **mất dữ liệu**!

| Đúng ✅ | Sai ❌ |
|---|---|
| Sign out máy `.30` → đăng nhập máy `.111` | Đăng nhập cả 2 máy cùng lúc |
| Làm việc trên 1 máy tại 1 thời điểm | Mở session RDP + console cùng lúc |

### 10.2. Phải Sign Out đúng cách

Profile chỉ được lưu lên server khi **Sign Out**, không phải khi:
- Tắt màn hình
- Lock máy (Win + L)
- Hibernate
- Tắt máy đột ngột

### 10.3. Yêu cầu kết nối mạng

| Tình huống | Roaming Profile |
|---|---|
| Cùng mạng LAN công ty | ✅ Hoạt động |
| WiFi công ty | ✅ Hoạt động |
| Laptop ở nhà + VPN | ✅ Hoạt động |
| Laptop ở nhà, không VPN | ❌ Không hoạt động |

### 10.4. Ứng dụng phải cài trên từng máy

Roaming Profile sync **dữ liệu và cài đặt** của ứng dụng, nhưng **không sync bản thân ứng dụng**. Ví dụ:
- Chrome phải cài trên cả 2 máy → bookmark, history sẽ được sync ✅
- Zalo phải cài trên cả 2 máy → tin nhắn, danh bạ sẽ được sync ✅
