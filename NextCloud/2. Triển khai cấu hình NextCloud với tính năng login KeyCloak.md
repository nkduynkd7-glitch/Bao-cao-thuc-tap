
# Triển khai Nextcloud tích hợp Keycloak (SSO) – Hướng dẫn chi tiết

## 1. Giới thiệu

Tài liệu này mô tả cách triển khai hệ thống **Nextcloud tích hợp đăng nhập SSO bằng Keycloak (OpenID Connect)**.

Mục tiêu:

- Người dùng đăng nhập **Nextcloud bằng Keycloak**
- Hệ thống **Single Sign-On (SSO)**
- Quản lý người dùng tập trung tại **Keycloak**
- Hỗ trợ tích hợp thêm các hệ thống khác như Mailcow, Gitlab, Grafana

---

## 2. Kiến trúc hệ thống

User → Nextcloud → Redirect OAuth2 → Keycloak → Authentication → Nextcloud session




## 3. Chuẩn bị môi trường

### Yêu cầu

- Ubuntu Server 22.04+
- Docker
- Docker Compose
- Domain public
- SSL (Let's Encrypt)

Ví dụ domain:

Nextcloud: cloud.example.com  
Keycloak: auth.example.com

---

## 6. Tạo Realm trong Keycloak

Admin Console → Create Realm

Tên:

nextcloud_realm

---

## 7. Tạo Client cho Nextcloud

Clients → Create

Client ID:

nextcloud

Client type:

OpenID Connect

Access type:

confidential

Redirect URI:

https://cloud.example.com/apps/sociallogin/custom_oidc/Keycloak

---

## 8. Lấy Client Secret

Clients → Credentials

Copy:

Client Secret

---

## 9. Cài app OIDC cho Nextcloud

Nextcloud → Apps → Search

Cài:

Social Login  
hoặc  
OpenID Connect Login

---

## 10. Cấu hình OpenID Connect

Nextcloud → Settings → Social Login

Authorize URL

https://auth.example.com/realms/nextcloud_realm/protocol/openid-connect/auth

Token URL

https://auth.example.com/realms/nextcloud_realm/protocol/openid-connect/token

UserInfo URL

https://auth.example.com/realms/nextcloud_realm/protocol/openid-connect/userinfo

Scope:

openid email profile

---

## 11. Mapper trong Keycloak

Clients → nextcloud → Mappers

Thêm:

email  
preferred_username  
name

## 14. Reverse Proxy ví dụ (Nginx)

server {
    server_name cloud.example.com;

    location / {
        proxy_pass http://localhost:8080;
    }
}

## 15. Security Best Practices

Khuyến nghị:

- Bật HTTPS
- Tắt local login Nextcloud
- Sử dụng MFA trong Keycloak
- Backup database
